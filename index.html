sliding-nightmare

carousel slider, developed 100% with vanilla javascript without external dependencies. It also uses the virtualization of elements which allows having a large number of elements without affecting the performance.

$ npm i sliding-nightmare --save

Examples

Basic usage

new SlidingNightmare('example-carousel-id');

Using options

It is recommended to use window.matchMedia to determine the number of elements visible according to the width of the device. In this example the images on small screens are distorted.

window.addEventListener('load', () => {
  new SlidingNightmare('options-example-carousel-id', {
    carouselStyles: {
      height: '120px',
     },
     cover: true,
     numOfVisibleElements: 4,
     animationDuration: 600,
     imagesPaddingVertical: '20px',
   });
});

Rendering HTMl Elements

Not only can you use images within the "nightmare-items" you can also implement any type of arrangement of elements, you just have to be careful with the styles that you use.

<div class="nightmare-item"> <div class="content-example"> <div class="red"> <ul class="list-group"> <li class="list-group-item">Lorem ipsum</li> <li class="list-group-item">Dolor sit amet</li> <li class="list-group-item">Consectetur adipisicing elit</li> </ul> </div> <div class="green"> <h3>Sliding Nightmare</h3> <p> Lorem ipsum dolor... </p> </div> </div> </div>
